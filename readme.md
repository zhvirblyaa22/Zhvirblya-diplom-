### **Корпоративная платформа для управления обучением ИТ-специалистов**



Жвирбля Анастасия Александровна, СДП-УИР-221

\-------------------------------------------------------------------------------------------------------------------------------------

ОПИСАНИЕ

\-------------------------------------------------------------------------------------------------------------------------------------

За основу разработки платформы была взята типовая конфигурация «1С:Анкетирование». Процесс реализации нацелен на создание функциональной и удобной подсистемы, которая сможет быть интегрирована в корпоративный контур 1С, способной организовать и управлять обучением с учётом требований к развитию сотрудников. Разработанный модуль для управления обучением IT-специалистов обеспечивает полный цикл учёта прохождения курсов обучения. Реализованы алгоритмы назначения обучения через механизм заданий и процедуры фиксации дат ознакомления с теорией и выполнения практики. Внедрены отчеты «Назначенные курсы» и «История обучения сотрудников» для аналитики. Разработанное решение автоматизирует учет обучения, обеспечивая целостность данных.



На пустую конфигурацию установлен каркас БСП согласно инструкциям на портале ИТС. 



В конфигурацию добавлена подсистема "Обучение", где находятся разработанные объекты. Подсистема выведена в командный интерфейс конфигурации, что означает, что она будет доступна пользователям системы и они смогут с ней взаимодействовать.



Разработан документ КурсыОбучения. На форме документа реализован механизм настройки курса, позволяющий пользователям с соответствующими ролями настроить содержательную часть курса, состав обучающихся и состав необходимых анкет к прохождению. Пользователи, имеющие роль за прохождение курса, получая доступ к этому же документу, благодаря реализованному механизму, на этой же форме могут в интерактивном режиме взаимодействовать с заранее настроенными блоками теории и практики, ставить отметки о прохождении курса с помощью специальных кнопок.



Документ интегрирован со стандартной подсистемой "Бизнес процессы и задачи", что позволяет ему являться предметом задания и задач. При старте курса реализована автоматическая отправка задания на пользователей, которые входят в состав обучающихся.



Данные о прохождении курса обучающимся хранятся в регистре сведений. Регистры сведений - специальный вид объектов конфигурации, предназначенных для хранения, более оптимизированной (быстрой) записью и получением данных из БД. Исходя из этого качества объекта, которое позволяет оптимизировать запросы к базе данных при проектировании отчетов (а отчеты строятся именно на запросах), а так же из-за того, что курс один, а обучающихся может быть несколько (каждый обучающийся при открытии курса должен видеть свой прогресс, а не прогресс кого-нибудь другого), хранение данных прогресса о прохождении курсов было решено реализовывать с помощью регистра сведений.



Для работы с системой и для доступа к объектам метаданных пользователям нужны соответствующие роли. Согласно инструкциям по внедрению стандартных подсистем "Анкетирование" и "Бизнес процессы и задачи" состав ролей был дополнен и подкорректирован для возможности работы соответствующих пользователей с курсами обучения (например: была создана группы и профили доступа для администрирования курсов и прохождения курсов, при добавлении пользователей в соответствующие группы доступа меняется принцип их работы с документом курса обучения).

КАК УСТАНОВИТЬ

\-------------------------------------------------------------------------------------------------------------------------------------



Для всех вариантов требуется установка платформы 1С:Предприятие. 

1С:Предприятие требует наличие приобретенных лицензий, дающих право на пользование программ фирмы 1С.



Для бесплатного ознакомления предоставляется учебный дистрибутив платформы (https://1c.by/v8/education\_products/programming\_set.php)



1\. Установка путем загрузки информационной базы:

\- Запустить 1С:Предприятие, в окне добавить/выбрать пустую конфигурацию;

\- Открыть конфигурацию в режиме "Конфигуратор";

\- В верхней панели управления выбрать "Администрирование" -> "Загрузить информационную базу" и выбрать файл с расширением ".dt".



2\. Установка путем загрузки конфигурации из файлов/файла (Не подходит для учебной версии платформы):

\- Открыть пустую конфигурацию в режиме "Конфигуратор";

\- В верхней панели управления выбрать "Конфигурация" -> "Загрузить конфигурацию из файлов", 

&#x20; в этом случае для загрузки следует указывать корневой каталог, содержащий файлы конфигурации в формате xml;



\- В верхней панели управления выбрать "Конфигурация" -> "Загрузить конфигурацию из файла", 

&#x20; в этом случае следует указать путь к файлу с расширением ".cf";



3\. Установка путем интеграции в существующую учетную систему (Не подходит для учебной версии платформы):

\- Открыть конфигурацию, в которую необходимо интегрировать модуль в режиме "Конфигуратор";

\- В верхней панели управления выбрать "Конфигурация" -> "Сравнить, объединить с конфигурацией из файла...";

\- В окне сравнения/объединения произвести слияние модуля с конфигурацией.
